## Tech-Distribution

Templates ohne HTML-Pages (React/Next.js/PHP/Frameworks) brauchen für Pattern-Extraktion eine andere Strategie (`.tsx` / `.blade.php` / `.vue` lesen statt `.html`). Für Phase 1 priorisieren wir HTML/Tailwind-Templates.

## Empfohlene Live-Demo-Sichtung (nächster Schritt)

Auswahl-Kriterien: Tailwind-Stack (näher an KOMPAGNON), Service-Industries (Heizung/Elektrik/Sanitär-Pattern übertragbar), reichlich HTML-Pages (mehr Pattern-Material).

## Workflow nach Sichtung

1. User postet Pattern-Notes pro Template (z.B. "Bricknet: Service-Comparison-Table sehenswert; Hero mit Sticky-Quote-Bar")
2. Claude leitet daraus Phase-1-Pattern-Kandidaten ab (HERO + CTA Bundle)
3. Pro Pattern: `unzip -j` der relevanten Page → Pattern-Identifikation → neutralisiertes HTML in `frontend/src/components/library/`
